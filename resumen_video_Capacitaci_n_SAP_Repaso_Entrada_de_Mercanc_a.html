Resumen Temático

Objetivo

Capacitar en el proceso de recepción de mercancías (entrada de mercancías) contra un pedido de compra en SAP S/4HANA utilizando la transacción MIGO, cubriendo escenarios como recepciones parciales, múltiples lotes, diferentes tipos de stock y traspasos internos.

Pasos del Proceso

  • Acceder a la transacción MIGO para realizar una entrada de mercancía contra un pedido.
  • Buscar y seleccionar el pedido de compra correspondiente.
  • Indicar la cantidad de material que se está recibiendo, permitiendo recepciones parciales.
  • Especificar el almacén de destino para el stock.
  • Registrar el lote del proveedor; el sistema genera un lote interno automáticamente.
  • Seleccionar el tipo de stock para la recepción (ej. libre utilización, control de calidad, bloqueado).
  • Utilizar el 'Indicador de Entrega Final' para cerrar la posición del pedido si no se esperan más entregas.
  • Verificar los datos y contabilizar el movimiento para generar un documento de material.
  • Realizar traspasos de stock entre diferentes estados (ej. bloqueado a libre utilización) o entre almacenes usando MIGO con la clase de movimiento adecuada.

Transacciones SAP

MIGOME23NMSC2NMMBEMB51

Puntos Clave

  • La transacción MIGO es central para todos los movimientos de mercancías, incluyendo recepciones y traspasos.
  • Es crucial diferenciar entre el lote interno generado por SAP y el lote del proveedor, que debe ser ingresado manualmente.
  • Las recepciones parciales son posibles y se gestionan controlando el 'Indicador de Entrega Final'.
  • El stock puede ser recibido en diferentes estados (libre utilización, bloqueado, control de calidad) según las necesidades del negocio.
  • Los traspasos de stock entre estados o almacenes no generan documentos contables, ya que no alteran el valor total del inventario.
  • La transacción ME23N permite visualizar el historial del pedido, que se actualiza con cada recepción de mercancía.
  • La transacción MSC2N permite modificar y enriquecer los datos del maestro de lotes después de su creación.

Recomendaciones

  • Utilizar la clase de valoración correcta en el pedido de compra para evitar errores durante la contabilización en MIGO.
  • Para una entrega física que contiene múltiples lotes, utilizar la función 'Repartir Cantidad' en MIGO.
  • Verificar los niveles y estados del stock antes y después de cada movimiento utilizando la transacción MMBE.
  • Utilizar el reporte MB51 para un análisis detallado de todos los movimientos de material históricos.
  • Asegurarse de que cada registro en MIGO corresponda a una recepción física real (ej. un camión, una descarga).
